## 2. 相关技术与理论基础

### 2.1 Flask框架

Flask是一个轻量级的Python Web应用框架，基于Werkzeug WSGI工具箱和Jinja2模板引擎。Flask的核心设计哲学是"微框架"，即保持核心简单但可扩展。

**核心概念：**

1. **路由（Routing）**：Flask使用装饰器`@app.route()`来定义URL路由，将URL映射到对应的视图函数。

   ```python
   @app.route('/')
   def index():
       return 'Hello, World!'
   ```

2. **模板（Templates）**：Flask使用Jinja2模板引擎，支持模板继承、条件判断、循环等功能，实现前后端分离。

3. **蓝图（Blueprint）**：蓝图用于组织大型应用的路由和视图函数，实现模块化开发。通过蓝图可以将应用拆分为多个模块，每个模块负责一部分功能。

   ```python
   from flask import Blueprint
   
   admin_bp = Blueprint('admin', __name__, url_prefix='/admin')
   
   @admin_bp.route('/')
   def dashboard():
       return 'Admin Dashboard'
   ```

4. **工厂函数（Application Factory）**：工厂函数是一种设计模式，用于创建Flask应用实例。通过工厂函数，可以在不同的配置环境下创建应用实例，便于测试和部署。

   ```python
   def create_app(config_class=Config):
       app = Flask(__name__)
       app.config.from_object(config_class)
       
       # 初始化扩展
       db.init_app(app)
       
       # 注册蓝图
       app.register_blueprint(admin_bp, url_prefix='/admin')
       
       return app
   ```

### 2.2 Flask-SQLAlchemy

Flask-SQLAlchemy是Flask的SQLAlchemy扩展，为Flask应用提供了SQLAlchemy的支持。SQLAlchemy是一个强大的Python SQL工具包和ORM框架，提供了高层次和低层次的数据库操作API。

**核心概念：**

1. **模型定义**：使用Python类定义数据库模型，每个类对应数据库中的一张表，类的属性对应表的字段。

   ```python
   from flask_sqlalchemy import SQLAlchemy
   
   db = SQLAlchemy()
   
   class Movie(db.Model):
       id = db.Column(db.Integer, primary_key=True)
       title = db.Column(db.String(200), nullable=False)
       category_id = db.Column(db.Integer, db.ForeignKey('category.id'))
   ```

2. **关系定义**：SQLAlchemy支持定义一对多、多对多等关系，通过`relationship()`实现。

   ```python
   class Category(db.Model):
       id = db.Column(db.Integer, primary_key=True)
       name = db.Column(db.String(50))
       movies = db.relationship('Movie', backref='category', lazy=True)
   ```

3. **数据库操作**：提供了方便的数据库操作API，包括查询、插入、更新、删除等。

   ```python
   # 查询所有电影
   movies = Movie.query.all()
   
   # 按条件查询
   movie = Movie.query.filter_by(title='Inception').first()
   
   # 插入数据
   new_movie = Movie(title='Inception', category_id=1)
   db.session.add(new_movie)
   db.session.commit()
   ```

### 2.3 Flask-WTF

Flask-WTF是Flask的WTForms扩展，提供了表单处理和验证功能，并集成了CSRF保护。

**核心概念：**

1. **表单定义**：使用Python类定义表单，每个表单字段对应一个类属性。

   ```python
   from flask_wtf import FlaskForm
   from wtforms import StringField, PasswordField, SubmitField
   from wtforms.validators import DataRequired, Length
   
   class LoginForm(FlaskForm):
       username = StringField('用户名', validators=[DataRequired(), Length(min=3, max=20)])
       password = PasswordField('密码', validators=[DataRequired()])
       submit = SubmitField('登录')
   ```

2. **表单验证**：WTForms提供了丰富的验证器，包括DataRequired、Length、Email、EqualTo等。

3. **CSRF保护**：Flask-WTF内置了CSRF保护功能，通过在表单中添加CSRF令牌，防止跨站请求伪造攻击。

   ```python
   from flask_wtf.csrf import CSRFProtect
   
   csrf = CSRFProtect()
   csrf.init_app(app)
   ```

### 2.4 Bootstrap框架

Bootstrap是一个开源的前端框架，提供了预定义的CSS样式和JavaScript组件，用于快速开发响应式网页。

**核心特性：**

1. **响应式网格系统**：Bootstrap使用12列网格系统，通过调整列的宽度和偏移，实现不同屏幕尺寸下的布局适配。

2. **预定义样式**：提供了按钮、表格、表单、导航栏等常用组件的预定义样式，开发者可以直接使用。

3. **JavaScript组件**：提供了模态框、下拉菜单、轮播图等交互组件，通过添加data属性或调用JavaScript方法即可使用。

在本项目中，使用Bootstrap 5实现前端页面的响应式布局，确保在不同设备（桌面、平板、手机）上都能良好显示。

```python
    app.config.from_object(config_class)
    
    # 初始化扩展
    db.init_app(app)
    csrf.init_app(app)
    login_manager.init_app(app)
    
    # 注册蓝图
    from app.main.views import main_bp
    from app.admin.views import admin_bp
    from app.user.views import user_bp
    app.register_blueprint(main_bp)
    app.register_blueprint(admin_bp, url_prefix='/admin')
    app.register_blueprint(user_bp, url_prefix='/user')
    
    # 创建数据库表
    with app.app_context():
        db.create_all()
        _seed_data(app)
    
    return app
```

**功能说明：** 工厂函数创建Flask应用实例，初始化扩展，注册蓝图，创建数据库表。通过工厂函数，可以在不同的配置环境下创建应用实例，便于测试和部署。

**如何保证正确性：** 通过`app.app_context()`创建应用上下文，确保数据库操作在正确的上下文中执行。

### 4.3 安全措施

#### 4.3.1 密码哈希存储

系统使用Werkzeug的`generate_password_hash()`和`check_password_hash()`函数实现密码哈希存储，避免明文存储密码。

**代码位置：** `app/models.py`

```python
from werkzeug.security import generate_password_hash, check_password_hash

class Admin(db.Model):
    """管理员模型"""
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    password_hash = db.Column(db.String(255), nullable=False)
    
    def set_password(self, password):
        """设置密码哈希值"""
        self.password_hash = generate_password_hash(password)
    
    def check_password(self, password):
        """验证密码"""
        return check_password_hash(self.password_hash, password)
```

### 5.3 分析讨论

#### 5.3.1 遇到的问题及解决方案

**问题1：SQLAlchemy模型关系定义错误**

在最初定义模型关系时，没有正确设置`backref`和`lazy`参数，导致查询关系时出现错误。

**解决方案：** 仔细阅读SQLAlchemy文档，正确设置`backref`和`lazy`参数。使用`backref`可以在关系的另一端自动创建反向引用，使用`lazy='dynamic'`可以控制关系的加载方式。

**问题2：WTForms表单验证失败**

在测试表单验证时，发现某些字段的验证器没有生效，导致非法数据被提交到数据库。

**解决方案：** 仔细检查表单类的定义，确保每个字段都添加了合适的验证器。在视图函数中，使用`form.validate_on_submit()`方法验证表单，确保只有验证通过后才处理数据。

**问题3：测试用例之间相互影响**

在运行单元测试时，发现某些测试用例会相互影响，导致测试结果不稳定。

**解决方案：** 使用`setUp()`和`tearDown()`方法为每个测试用例创建独立的环境。在`setUp()`中创建应用上下文和数据库表，在`tearDown()`中清理数据库和上下文。
